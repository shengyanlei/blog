# 实施计划（全栈）

## 目标
在现有代码基础上补齐博客的基础功能，打通后端 API 与前端页面，确保可登录、可发文、可浏览、可评论。

## 工作拆分
1) 后端：修复乱码与语法问题，补充按 slug 查询文章接口，增加全局 CORS，默认管理员种子数据。
2) 前端基建：创建 Axios 封装与类型定义，处理 Token 注入、错误提示与加载态。
3) 公共站点：文章列表/搜索、文章详情、评论列表与提交对接后端；UI 微调保持现有设计。
4) 管理端：登录对接、仪表盘数据拉取，补充文章管理（列表 + 新建/发布）基础流程。
5) 验证：构建/静态检查并给出手动验证指南。

## 交付标准
- 后端可编译、接口涵盖文章/分类/标签/评论/认证；跨域可用；存在默认管理员。
- 前端页面真实消费后端数据，核心流程（浏览、评论、登录、发文）可闭环。
- 文档备注运行与验证方式。

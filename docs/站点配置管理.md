# 站点配置管理（application.yml）

## 目标
将博客的核心品牌信息与关于页内容统一集中在一个可编辑文件中，避免分散在多个页面组件里重复维护。

配置文件路径：

- `apps/spa/public/application.yml`

## 生效方式

前端启动后会运行时请求：

- `/application.yml`（`cache: no-store`）

如果 YAML 文件不可访问、语法错误或字段缺失，系统会自动回退到内置默认配置，不会导致页面白屏。

## 可配置项

### 1) `site.meta`

- `title`：浏览器标题（`document.title`）
- `subtitle`：站点副标题（当前预留字段）

### 2) `site.brand`

- `navName`：顶部导航左侧站点名
- `heroEyebrow`：首页主视觉上方小标题
- `heroTitle`：首页主标题
- `heroDescription`：首页主描述
- `footerText`：页脚文案
- `navLabels.home/about/archive/admin`：导航与后台按钮文本

### 3) `site.profile`

- `name`：作者名
- `initials`：作者缩写（头像占位文字）
- `role`：角色描述
- `bio`：作者简介
- `location`：所在地/工作方式
- `expertise`：擅长领域
- `email`：联系邮箱
- `tags`：标签数组
- `avatarUrl`：作者头像地址

### 4) `about`

- `heading`：关于页主标题
- `intro`：关于页开场介绍
- `ctaArchive`：关于页按钮（文章）
- `ctaContact`：关于页按钮（联系）
- `focusTitle`：方向模块标题
- `focusAreas[]`：方向卡片列表（`title` / `description`）
- `principlesTitle`：工作方式标题
- `principles[]`：工作方式列表
- `nowTitle`：最近在做标题
- `nowList[]`：最近在做列表
- `timelineTitle`：时间线标题
- `timeline[]`：时间线列表（`year` / `title` / `note`）

## 示例

```yaml
site:
  meta:
    title: "碎念随风"
    subtitle: "设计、前端与写作"
  brand:
    navName: "碎念随风"
    heroEyebrow: "Glacier Log"
    heroTitle: "霜蓝札记"
    heroDescription: "在设计、代码与慢旅行之间，收集观察与技术笔记。"
    footerText: "© Antigravity 提供技术支持"
    navLabels:
      home: "首页"
      about: "关于"
      archive: "文章"
      admin: "后台管理"
```

## 部署缓存建议（Nginx）

建议为 `/application.yml` 关闭强缓存，避免修改后前端读取旧内容：

```nginx
location = /application.yml {
    add_header Cache-Control "no-store, no-cache, must-revalidate, max-age=0";
}
```

# 后台账号安全改造（上云执行）

## 目标
- 生产环境禁用默认弱口令管理员。
- 生产环境关闭公开注册接口 `/api/auth/register`。
- 使用一次性 SQL 创建或更新 `shyl` 管理员账号。

## 代码开关
- `backend/src/main/resources/application-dev.yml`
  - `app.seed.default-admin-enabled: true`
  - `app.auth.public-register-enabled: true`
- `backend/src/main/resources/application-prod.yml`
  - `app.seed.default-admin-enabled: false`
  - `app.auth.public-register-enabled: false`

## 生产执行步骤（Linux 示例）
1. 启用生产 profile：
```bash
export SPRING_PROFILES_ACTIVE=prod
```

2. 生成 bcrypt 哈希（明文密码不入库、不进仓库）：
```bash
cd /path/to/blog/backend
export ADMIN_PASSWORD='replace-with-your-own-strong-password'
mvn -q -DskipTests org.codehaus.mojo:exec-maven-plugin:3.5.0:java \
  -Dexec.mainClass=com.blog.tools.BcryptHashGenerator \
  -Dexec.args="$ADMIN_PASSWORD"
```

3. 执行一次性管理员 SQL：
- 打开 `backend/scripts/prod-admin-upsert.sql`
- 将 `__BCRYPT_HASH__` 替换为上一步输出的哈希
- 在生产库执行该 SQL

4. 启动后验证：
```bash
curl -i -X POST http://127.0.0.1:8080/api/auth/register \
  -H 'Content-Type: application/json' \
  -d '{"username":"probe_user","email":"probe@example.com","password":"ProbePass@123"}'
```
- 期望：`HTTP/1.1 403`（prod）

## 注意
- 不要把明文密码或固定哈希提交到 Git。
- `admin` 弱口令账号在生产环境应删除或禁用（SQL 模板默认删除）。

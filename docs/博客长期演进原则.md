# 博客长期演进原则

本项目面向长期内容运营，不以短期演示效果为目标。任何与内容展示相关的功能迭代，都需要默认假设文章数量会持续增长，并在设计与实现阶段提前为增长留出结构空间。

## 核心原则

1. 首页任何内容位都必须有明确的容量上限，不能无限堆叠。
2. 内容增长后必须通过分组、分页、换页、筛选或截断控制首屏信息密度。
3. 首页、归档页、专题区等高频浏览页面，默认按“长期运营站点”标准设计，而不是按“少量内容展示页”设计。
4. 精选、待阅、最新、分类归档等模块必须避免重复展示同一批内容，保证浏览节奏清晰。
5. 当内容量增大时，优先保证可发现性、可维护性和视觉秩序，而不是继续增加首屏元素数量。

## 实施要求

- 新增首页内容模块时，必须同时定义数据来源、展示上限和超量后的处理方式。
- 新增归档与列表页时，必须说明在“文章很多”的情况下如何分页、分组或筛选。
- 任何默认展示全部内容的方案，都必须在评审时说明为什么不会在长期运营中失控。
- 如果一个模块未来存在明显增长风险，应在首次实现时就提供可扩展的查询参数和 UI 容量控制。

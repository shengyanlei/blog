# 素材池与账号管理一期

## 1. 背景与目标
- 将素材池从足迹管理中独立成后台左侧单独菜单，降低操作复杂度。
- 素材除绑定足迹外，新增绑定文章封面能力，支持单封面替换。
- 增加账号管理，支持按账号配置可见菜单，实现不同角色不同后台视图。
- 本文档是本期唯一需求基线，后续实现与验收均以本文为准。

## 2. 角色与权限模型
- 角色仅包含 `OWNER` 与 `MEMBER`。
- `OWNER` 当前固定为 `username=shyl`，不可删除、不可禁用、不可降权。
- `MEMBER` 的后台菜单由主人动态授予，未授权菜单不展示。
- 账号管理仅主人可见可用，成员不可访问。
- 本期授权粒度为菜单可见性，不做接口动作级授权矩阵。

## 3. 菜单信息架构
- 菜单新增：`素材池`、`账号管理`。
- 菜单拆分：`足迹管理`去除素材页签，仅保留旅程与计划。
- `OWNER` 默认展示全部后台菜单。
- `MEMBER` 根据授权菜单展示。
- 无可见菜单的账号登录后进入无权限占位页。

## 4. 素材池功能需求
- 素材池提供独立页面，支持上传、待处理筛选、地址绑定、归档到旅程。
- 素材仍复用现有 `footprint_photo` 数据模型，避免重复存储。
- 新增素材删除能力：被文章封面或足迹地址引用时禁止删除。
- 删除失败返回冲突状态并提示引用来源（文章/足迹）。

## 5. 文章封面绑定需求
- 文章新增 `coverPhotoId` 字段，引用素材池图片。
- 每篇文章仅允许一个封面，支持替换与清空。
- 文章详情与列表返回 `coverPhotoId` 和 `coverImage`。
- 前台文章卡片与详情页沿用 `coverImage` 展示逻辑。

## 6. 账号管理需求
- 提供账号列表、创建、编辑、重置密码、禁用、删除能力。
- 创建账号时可直接分配菜单权限。
- 仅允许创建/维护 `MEMBER` 账号。
- 非主人账号不允许在设置中心自助改密。
- 主人可动态修改其他账号密码与菜单授权。

## 7. 数据模型与迁移
- `users` 表：`role` 统一为 `OWNER|MEMBER`，新增 `enabled`。
- 新增 `user_tab_permission` 表：`user_id + tab_code` 唯一。
- `articles` 表新增 `cover_photo_id`，关联 `footprint_photo.id`。
- 迁移规则：
  - `shyl -> OWNER`
  - 其他账号 -> `MEMBER`
  - `shyl` 自动具备全量菜单权限
  - 其他账号默认无菜单权限

## 8. API 契约
- 登录响应新增：
  - `user.enabled`
  - `user.tabCodes`
- 文章请求/响应新增：
  - `coverPhotoId`
  - `coverImage`
- 新增账号管理接口：
  - `GET /api/admin/accounts`
  - `POST /api/admin/accounts`
  - `PUT /api/admin/accounts/{id}`
  - `PUT /api/admin/accounts/{id}/password`
  - `DELETE /api/admin/accounts/{id}`
  - `GET /api/admin/accounts/permission-options`
- 新增素材命名空间：
  - `/api/materials/*`
- 兼容保留：
  - `/api/journeys/assets/*`

## 9. 测试与验收
- `shyl` 登录后显示全部菜单，账号管理可访问。
- 新建成员并仅授权足迹与素材，登录后仅显示两项菜单。
- 成员可完成素材上传、绑定地址、归档旅程操作。
- 文章可绑定素材封面并在前台展示。
- 素材被文章或足迹引用时删除返回 `409`。
- 主人可重置成员密码；禁用成员后成员登录失败。
- 对 `shyl` 的删除/禁用/降权操作必须失败。

## 10. 上线与回滚
- 上线顺序：数据库迁移 -> 后端 -> 前端。
- 发布后核验：登录、菜单、素材、文章封面、账号管理全链路。
- 回滚策略：
  - 前端回滚静态资源
  - 后端保留旧素材接口兼容
  - 新增数据库字段保留，不做破坏性回滚
